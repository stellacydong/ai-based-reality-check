Motivations

IntroductionWhen using social networks, have you ever encountered a 'fake identity'? 

Anyone can create a fake profile image using image editing tools, or even using deep learning based generators.
If you are interested in making the world wide web a better place by recognizing such fake faces, you should check this dataset. What's Inside and WhyOur dataset contains expert-generated high-quality photoshopped face images.
The images are composite of different faces, separated by eyes, nose, mouth, or whole face.

You may wonder why we need these expensive images other than images automatically generated by computers.
Say we want to train a classifier for real and fake face images. 

In case of generative models like Generative Adversarial Networks (GAN), it is very easy to generate fake face images.
Then, a classifier can be trained using those images, and they do great job discriminating real and generated face images.

We can easily assume that the classifier learns some kind of pattern between images generated by GANs.
However, those patterns can be futile in front of human experts, since exquisite counterfeits by experts are created in completely different process. 

Why: 
With the growth of social media, there has been an increased growth in creation and distribution of altered images. These images could potentially be detrimental towards the self-esteem of middle-aged individuals who often use these social media platforms. This application is meant to detect any images that may have been tampered with by any applications. The intended effect of this is to hopefully break the social expectations of perfection that is so commonly 

Who:
This application can be sold to young and middle-aged people (teenagers to people in their mid-20’s), people who are most active on social media. They are the most likely to use the application. 

Dataset

There are two datasets that we chose to use. The first one whose fake images are made by Photoshop Experts and is from Computational Intelligence and Photography Lab at Department of Computer Science, Yonsei University, and the second dataset consists all 70k REAL faces from the Flickr dataset collected by Nvidia, and 70k fake faces sampled from the 1 Million FAKE faces (generated by StyleGAN) that was provided by Bojan. Hence in total, we have 142,041 images. 

Dataset 1


There are 2 classes in the first dataset: real face vs. fake face. In this dataset, there are real faces, which are unchanged and unedited pictures of faces, and “fake” faces, which have filters or have been edited. In the fake one, there are easy, middle, hard levels (see Figure above). There are 2041 images in total, where 1081 of them are real images, and 960 fake images. Each image is 224 x 224 x 3 pixel representation. 

Dataset 2

The second dataset has 140k images in total for 2 classes: real face vs. fake face. It consists of all 70k REAL faces from the Flickr dataset collected by Nvidia, as well as 70k fake faces sampled from the 1 Million FAKE faces (generated by StyleGAN) that was provided by Bojan. 

Convolutional Neural Networks 
on the first dataset

We started with the first dataset, which 2041 images in total, where 1081 of them are real images, and 960 fake images. We thought that the dataset is complete so we started building and training the models. ​The first model we built was a 13-layer Convolutional Neural Network (CNN) as shown below. 

We trained the model on this dataset for 15 epochs. The loss function is binary crossentropy. We use Adam method as optimizer with learning rate 0.0001. The accuracy and loss for the training is shown below. During the training the best accuracy for the validation set is 58%, and after the training, the accuracy on the test set is 65.23%. 

VGG16 on the first dataset

The second model we built was a modified version of VGG16.

The VGG16 convolutional neural network architecture has 16 layers, using weights that are already trained previously on ImageNet. This model is able to achieve a 92.7% accuracy, ImageNet which has over 14 million images belonging to 1000 classes. The architecture of the original VGG16 model is shown below: 

In order to input an image in the algorithm, the input images need to be resized to 224x244x3. Each convolution layer has a 3x3 filter with a stride of 1 and a 2x2 filter with a stride of 2. These layers help extract the important information from the image and prepare the image for the next layer. There is also a pooling layer, whose job is to make the image have less parameters to be computed on. There is also a fully connected layer which connects to the previous layers.The activation function used for this algorithm is softmax. Image below:

The accuracy for the test set is 59.27%.

DenseNet121 on the union of the first and second dataset

After building 2 CNNs, we saw pretty low accuracy on the test set for both models above, so we realized that we needed to collect more images. We found the second dataset and combined both datasets. The total number of images for both datasets is 142041. The distribution of 2 classes (fake vs. real face images) is shown below, and it indicates that the entire dataset is balanced. 

We divided the entire dataset into 3 parts: the training set which has 113632 images, the validation set which has 9469 images, and the test set which has 18940 images.

We applied modified version of the DenseNet121 model on this dataset, which binary-crossentrop as loss function, and adam method as optimizer. 

We chose this Densely Connected Convolutional Networks model because DenseNets have several compelling advantages: they alleviate the vanishing-gradient problem, strengthen feature propagation, encourage feature reuse, and substantially reduce the number of parameters.

The difference between DenseNet and VGG is that DenseNet has one additional layer, “Transition layer” which prevents the gradient vanishing problems and reduces the complexity of the algorithm. 

The accuracy of the DenseNet is 96% for the test set.
